Migrating from 1.0 to 1.1

Unfortunately, we had to change a number of things in 1.1 that are not compatible with the 1.0 release. It is our belief that each of these changes are for the better. Besides, we wanted to establish a good foundation as early as possible and continuing with a shaky 1.0 structure was counter to that philosophy.

To make a long story short, we've compiled a list of the steps you'll need to take to move to the 1.1 release if you're currently using 1.0.

XML Format

Version 1.0 (OLD)

<?xml version="1.0"?>
<list>
  <item value="1">Record 1</item>
  <item value="2">Record 2</item>
  <item value="3">Record 3</item>
</list>

Version 1.1 (NEW)

<?xml version="1.0" encoding="UTF-8"?>
<ajax-response>
  <response>
    <item>
      <name>Record 1</name>
      <value>1</value>
    </item>
    <item>
      <name>Record 2</name>
      <value>2</value>
    </item>
    <item>
      <name>Record 3</name>
      <value>3</value>
    </item>
  </response>
</ajax-response>

You should notice two things. The structure of the XML is fundamentally different. We're no longer using the value attribute, replacing it with a value element. This is simply a matter of flexibility. For instance, you can't put a CDATA element inside an attribute.

The second thing you'll notice is not necessarily a requirement, but a recommendation. Notice the encoding attribute of the XML declaration. You should get into the habit of specifying the encoding for the XML returned to the client. We talk about this in our FAQ.

The exception to this is that the ajax:htmlContent tag does not require an XML response. It expects an HTML response instead and the AJAX function will not parse it as XML; it will simply insert the content of the response as is.

Note: There is a helper class to assist in building the XML if you don't want to do it by hand...see the Advanced Usage section for more information.

JSP View Changes

The 1.1 release relies on the Prototype framework. You should now include the prototype.js file in all pages using the Ajax JSP Tags.

<script type="text/javascript" src="prototype-1.3.1.js"></script>
<script type="text/javascript" src="ajaxtags-1.1.5.js"></script>

There is no longer any need to specify an autocomplete="off" attribute on the form field where the user types. This is now taken care of in the JavaScript code.
