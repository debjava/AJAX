Installation

Requirements

To use the tag library, you'll need the following:

  • JDK 1.5+
  • Servlet container running Servlets 2.3+ and JSP 1.0+ (Note: sample application requires Servlet 2.4 and JSP 2.0)
  • Copy of the Prototype framework. Currently, we depend upon version 1.5.0.
  • Copy of the Scriptaculous library. Currently, we depend upon version 1.7.0.
  • Copy of the OverLIBMWS library.

If you intend to build from the source, you'll need the following:

  • Maven 2
  • source dependencies

Installation and Set Up

Step 1: Copy AjaxTags jar file

Copy the ajaxtags-1.3.jar into your WEB-INF/lib directory.

Step 2: Define TLD Reference

Note: This step is not necessary for Servlet 2.4 containers.

For JSP 1.x users, copy the ajaxtags.tld file to your WEB-INF directory and include a taglib reference in your web.xml file. ajaxtags.tld is located inside the META-INF folder in ajaxtags-1.3.jar

<taglib>
  <uri>http://ajaxtags.org/tags/ajax</uri>
  <location>/WEB-INF/ajaxtags.tld</location>
</taglib>

Step 3: Create Server-side Handler

There must be a server-side handler that will process the AJAX request. This could be a servlet, Struts action, another JSP, or even a non-Java, server-side component. The only requirement of this component is that it return an XML file that represents a list of results based on the query passed to it from the client-side AJAX functions.

There is a helper class to assist in building the XML if you don't want to do it by hand...see the Advanced Usage section for more information.

XML Format

<?xml version="1.0" encoding="UTF-8"?>
<ajax-response>
  <response>
    <item>
      <name>Record 1</name>
      <value>1</value>
    </item>
    <item>
      <name>Record 2</name>
      <value>2</value>
    </item>
    <item>
      <name>Record 3</name>
      <value>3</value>
    </item>
  </response>
</ajax-response>

Step 4: Declare TLD Reference on JSP

For each JSP in which you wish to use the AJAX tag, simply define it as with any other tag library.

<%@ taglib uri="http://ajaxtags.org/tags/ajax" prefix="ajax" %>

Step 5: Include JavaScript Core

With AJAX, you'll need JavaScript. Fortunately, everything you'll need is contained in two files. The first is ajaxtags-1.2.js file, while the second is the prototype-1.3.1.js file, on which the AJAX tags depend. Just include them in your project and declare the location using the following as an example. It's best to include this in the head. Note: you MUST include the prototype.js file BEFORE the ajaxtags.js file.

Incidentally, the reason why a (bulky) JavaScript source file was chosen versus having each JSP tag generate only what it needs is because it's assumed that most browsers cache external resources like stylesheets, images, and JavaScript files. This should increase performance slightly. Besides, this core JavaScript is generic enough to use for other purposes that the tag library does not yet provide.

  <script type="text/javascript" src="<%=request.getContextPath()%>/js/prototype.js"></script>
  <script type="text/javascript" src="<%=request.getContextPath()%>/js/scriptaculous/scriptaculous.js"></script>
  <script type="text/javascript" src="<%=request.getContextPath()%>/js/overlibmws/overlibmws.js"></script>
  <script type="text/javascript" src="<%=request.getContextPath()%>/js/overlibmws/overlibmws_crossframe.js"></script>
  <script type="text/javascript" src="<%=request.getContextPath()%>/js/overlibmws/overlibmws_iframe.js"></script>
  <script type="text/javascript" src="<%=request.getContextPath()%>/js/overlibmws/overlibmws_hide.js"></script>
  <script type="text/javascript" src="<%=request.getContextPath()%>/js/overlibmws/overlibmws_shadow.js"></script>
  <script type="text/javascript" src="<%=request.getContextPath()%>/js/ajax/ajaxtags.js"></script>

Step 6: Define CSS Styles

Many tags rely heavily on CSS to help visually represent their function. For example, in the case of the codeajax:autocomplete/code tag, CSS transforms a simple unordered list (i.e., UL > LI tags) into a dropdown list.

While the content is fairly constrained, it is basically up to the user to define the exact CSS s/he would like to use. The example application provides a fairly good starting point (if we do say so ourselves). A sample file is also included in the main project's distribution. Still, you CSS gurus will likely find more visually pleasing routes to success.

  <link rel="stylesheet" type="text/css" href="<%=request.getContextPath()%>/css/ajaxtags.css" />
  <link rel="stylesheet" type="text/css" href="<%=request.getContextPath()%>/css/displaytag.css" />
